# Communication Protocols

Di dunia AIoT, perangkat tidak bisa jalan sendiri. Sensor baca data, mikrokontroler mengirim data, server memproses, lalu dashboard menampilkan hasilnya. Supaya alurnya rapi dan semua perangkat "ngobrol" dengan bahasa yang sama, kita butuh protokol komunikasi.

Sederhananya, protokol adalah aturan main pertukaran data antara pengirim dan penerima. Mulai dari format data, cara kirim, sampai cara membalas. Di bagian ini, kita fokus ke empat protokol yang sering dipakai di AIoT:

- HTTP untuk pola request-response berbasis web.
- MQTT untuk komunikasi real-time berbasis publish-subscribe.
- WebSocket untuk koneksi dua arah yang persisten.
- CoAP untuk komunikasi ringan pada perangkat dan jaringan terbatas.
